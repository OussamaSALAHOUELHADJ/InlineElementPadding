Element in html are more fun, they have a lot of different properties, from what we define like style, class, id and more, to some really "inhuman" properties like rendering, view and a LOT more of bits and bytes!

In this part, we will focus only on getting the height.

MDN in this article titled: Determining the dimensions of elements defines several ways to get the dimension of an element via the properties of this element, which leads us to another sub-problem: In the exam question wich dimension are meant, considering the several ones that exist:

  • The offset (offsetHeight and offsetWidth) as they define the "How much room does it use up?".
  • The client (clientHeight and clientWidth) as they define the "What's the size of the displayed content?"
  • The scroll (scrollHeight and scrollWidth) as they define "How big is the content?"

without mentioning the calculation by using the viewport or page offset...ect. For more details MSDN: Measuring Element Dimension and Location.

Lemma: it is enough to prove that exists a way to find inline element dimensions with padding to prove that padding adds dimensions to the element.

1) How much room does it use up? : the offset

MDN defines offsetHeightas: The HTMLElement.offsetHeight read-only property returns the height of an element, including vertical padding and borders, as an integer.

2) What's the size of the displayed content? : clientHeight

MDN defines clientHeightas: The Element.clientHeight read-only property is zero for elements with no CSS or inline layout boxes; otherwise, it's the inner height of an element in pixels. It includes padding but excludes borders, margins, and horizontal scrollbars (if present).
clientHeight can be calculated as: CSS height + CSS padding - height of horizontal scrollbar (if present).
